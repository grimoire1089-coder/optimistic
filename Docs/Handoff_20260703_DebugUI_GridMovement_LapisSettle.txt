  - 行動モジュール側の移動はまだ旧方式が多く、通常散歩だけグリッドステップ化しても完全には揃わない。

5. 直近ユーザーの考え/設計方針
------------------------------------------------------------

ユーザーの指摘:
- 今の移動方式がグリッドのズレを呼んでいる。
- 1コマ/1ターン/1グリッド、たぶん48px単位できっちり動かす形にした方がいい。
- 移動してすぐ行動するからズレるのもあると思う。
- 移動を終えた後に完全に静止する時間を作って、そのあと行動させた方が良い。

現時点の実装方針:
- 通常散歩はグリッドステップ移動化済み。
- 座る/ラピスは行動前静止待機を入れた。
- ただし、水分補給・シャワー・クラフト・入口移動・睡眠前移動はまだ旧方式が残っている可能性が高い。

次の推奨作業:
1. Godotで最新mainを開き、まずパースエラー確認。
2. ロビンを動かしてログ確認。
   - sitting / lapis_standing の前に settling=true が出るか。
   - ラピス開始時/終了時にposがグリッド中心へ戻るか。
3. 通常散歩で斜めっぽく見える原因を確認。
   - 経路自体は上下左右のみのはず。
   - 開始時の位置ズレ補正が必要なら、RobinRandomWanderModule._pick_next_grid_path() で start_cell がwalkableでも現在セル中心へスナップ/ステップする処理を入れる。
4. 行動前移動の共通化。
   - 水分補給: AICharacterHydrateBehaviorModule.gd
   - シャワー: AICharacterHygieneBehaviorModule.gd
   - クラフト: AICharacterCraftBehaviorModule.gd
   - 入口移動: AICharacterEntranceTravelBehaviorModule.gd
   - 睡眠前移動: AICharacterSleepBehaviorModule.gd
   - 座る/ラピス: AICharacterSitBehaviorModule.gd
5. 最終的には共通モジュール化推奨。
   - 例: Scripts/Characters/Modules/AICharacterGridStepMovementModule.gd
   - 各AI行動モジュールが直接 velocity を返すのではなく、target_cell / next_cell を渡して1グリッド単位移動に寄せる。

6. 既知のリスク/注意点
------------------------------------------------------------

1. AICharacterActionItemDisplayModule.gd にラピス補正が入っている
- 本来は表示モジュールに移動補正を入れるのは少し設計的に微妙。
- 急ぎの安全対策として入れたもの。
- Codexで作業するなら、最終的には SitBehaviorModule か共通グリッド移動モジュール側へ移すのが綺麗。

2. AICharacterLapisGridSnapModule.gd は未接続の可能性あり
- 使うならシーンへ追加。
- 使わないなら削除か、後で共通移動モジュールへ統合。

3. RobinWanderActor.gd の大規模更新は注意
- 一度既存関数を消してしまう事故があった。
- 必ず差分で編集し、クリック/SFX/選択処理を残す。

4. move_and_slide() と直接position移動の混在
- 通常散歩は直接position補間。
- 行動モジュールの多くは velocity + move_and_slide()。
- 混在がズレ原因になりやすい。
- 方針としては、AI移動全体をグリッドステップ方式へ寄せた方が良い。

5. グリッドサイズ
- RoomMapGridModule.gd の cell_size は Vector2(48, 48)。
- ユーザーも48px想定で話している。
