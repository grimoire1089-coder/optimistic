# Character Footprint Design

## 基本方針

ロビン達AIキャラクターは、見た目・物理衝突・行動処理で使うサイズを分ける。

| 用途 | 基準サイズ | 目的 |
| --- | --- | --- |
| グラフィック / 行動処理 | 横2グリッド x 縦4グリッド | 家具との使用判定、睡眠判定、行動対象との位置関係、見た目上の重なり判定 |
| 物理衝突 / 移動時の当たり判定 | 下半身側の横2グリッド x 縦2グリッド | 家具や壁に対する実体の当たり判定。頭や上半身は家具に重なってもよい |

## 重要な注意

`actor_grid_footprint` のようなAI行動モジュール側の footprint は、原則としてキャラクターグラフィック全体の横2 x 縦4を基準にする。

一方で、`CollisionShape2D` や `AICharacterFootprintShapeModule` のような物理判定・実体判定は、下半身側の横2 x 縦2を基準にする。

つまり、次のように扱う。

- キャラがベッドや家具を使えるかどうかは、見た目の2x4グリッドを基準に判断する。
- 実際に家具へぶつかって止まるかどうかは、下半身の2x2グリッドだけを見る。
- 頭や上半身が寝具・机・キッチンなどにめり込んで見えるのは許容する。
- 足元だけでAI行動判定を作ると、見た目では十分届いているのに「まだ届いていない」扱いになりやすい。

## 睡眠処理の考え方

寝具へ向かう場合、物理的には下半身だけが衝突する。
ただし、睡眠開始判定はキャラクター全体の見た目を基準にする。

そのため、以下のどちらかを満たしたら寝具睡眠を開始してよい。

1. 行動処理用の2x4範囲が寝具使用範囲に到達した。
2. キャラクターの見た目矩形が寝具のグリッド矩形に重なった。

## 今後の実装ルール

AI行動モジュールを追加・修正する時は、以下を混同しないこと。

- `actor_grid_footprint` は処理用。基本は `Vector2i(2, 4)`。
- 物理用の形状や足元判定は別プロパティで扱う。
- 「詰まるから処理用 footprint を小さくする」のは避ける。
- 詰まり対策は、物理判定・直接グリッド移動・見た目重なり判定で解決する。

## 現在のロビン基準

- グラフィック: 横2 x 縦4
- 物理衝突: 下半身 横2 x 縦2
- 睡眠・水分補給・制作・マップ移動などの行動処理: 横2 x 縦4
