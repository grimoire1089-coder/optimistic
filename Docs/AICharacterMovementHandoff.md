# AI Character Movement Handoff

作成日: 2026-07-02

このメモは、GPTからの引継ぎメモ `codex_handoff_ai_movement_20260702.txt` のうち、今後の実装で守るべき方針をプロジェクト内に残すための要約。

## 守るべき方針

- 関係ないコードは消さない。
- キャラクター本体スクリプトは本体の責務に留め、機能はモジュール形式を維持する。
- 既存の行動モジュールを全部一気に置き換えない。
- 新しい共通モジュールを作る場合も、既存モジュールから少しずつ移行する。
- 変更は、まず対象の行動だけに絞って最小修正で見る。
- 水分補給・睡眠・制作・マップ移動などの移動処理は、ログとハイライトで確認できる状態を保つ。

## Footprint の絶対ルール

- AI行動処理用の `actor_grid_footprint` は原則 `Vector2i(2, 4)`。
- 物理衝突・実体判定は下半身 `Vector2i(2, 2)`。
- 詰まり対策として `actor_grid_footprint` を小さくしない。
- 頭や上半身が寝具・キッチン・机などに重なって見えるのは許容する。
- 足元だけで行動判定を作ると、見た目では届いているのにまだ届いていない扱いになりやすい。

詳細は `Docs/CharacterFootprintDesign.md` を参照する。

## 移動処理の実装方針

- 行動開始時にターゲット家具と使用セルをできるだけ固定する。
- 行動中に毎フレーム家具候補を再探索しない。
- `_path_cells` が残っている間は経路を作り直さない。
- 家具が消えた、移動した、footprint が変わった、target cell が無効になった場合だけ再探索する。
- 家具候補ごとに使用セルとパススコアを何度も計算する処理は重くなりやすいので注意する。

## デバッグ表示とログ

- `ai_movement_behavior` グループに入っている active な移動モジュールから経路情報を読む。
- 経路セル、次セル、最終目標セル、現在の処理用 footprint を見える状態にする。
- `AICharacterMovementDebugModule.gd` の `plan=` では、可能なら `get_debug_movement_summary()` を優先する。
- 移動中のログでは `target_cell`、`next_cell`、`path`、`footprint` が確認できるようにする。
- 移動が終わった状態や座っている状態などでは、不要な経路ハイライトを残さない。

## 今後の改善候補

- 移動経路計算を共通モジュール化する。
- 家具使用セルとパススコアをキャッシュする。
- `FurniturePlacementModule` などに家具配置の `layout_version` を持たせ、キャッシュ無効化に使う。
- 直接グリッド移動と `move_and_slide()` による物理移動の使い分けを整理する。
- ただし、これらは一気に置き換えず、水分補給や睡眠など安定している範囲から少しずつ移行する。
