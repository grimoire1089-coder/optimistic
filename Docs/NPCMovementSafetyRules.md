# NPC移動の安全ルール

## 現段階の制限

今は、マップ上で同時に動くAIキャラクターを1人までにする。

理由:

- 経路探索の原因切り分けを簡単にする
- AI同士の重なりを避ける
- 家具判定とAI判定を混ぜても安全に確認できるようにする
- NPCが増えた時に重くならないよう、先に小さく試す

## グリッド候補

家具が置かれているグリッドと同じように、AIキャラクターが立っているグリッドも経路候補から外す。

今は全AIを `2x4` footprint として扱う。
将来的にキャラクターごとのサイズが必要になったら、各AIに `get_actor_grid_footprint()` を持たせて拡張する。

## 実装

- `AICharacterMovementCoordinator.gd`
  - 動けるAIを1人に制限する軽量管理モジュール。
- `AICharacterRandomWanderModule.gd`
  - ロビンのランダム移動を継承した汎用移動モジュール。
  - 家具に加えて、AIキャラクターのグリッドも避ける。
- `ZippyActor.gd`
  - 実行時に汎用移動モジュールを生成して接続する。

## 次の確認項目

- ジッピーがグリッド単位で動くこと。
- ロビンが移動中ならジッピーが待機すること。
- ジッピーがロビンのいる2x4グリッドを目的地にしないこと。
- POS表示で赤枠が2x4のまま保たれること。
