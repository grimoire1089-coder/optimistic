# AI Character Adopted Extra Systems Spec

作成日: 2026-07-08

この文書は、AIキャラクター生活シムに追加採用する仕様を記録するためのメモである。

コード作業はまだ行わない。

## 採用する追加システム

以下を将来仕様として採用する。

```text
1. 行動理由ログ
2. AIキャラクターの習慣
3. 部屋・施設の雰囲気タグ
4. 放置復帰レポートの要約ランク
5. 常連システム
6. 小さな流行
7. 変化チップ安全装置
8. 安心して終了できる確認パネル
```

特に以下2つは必要仕様として扱う。

```text
7. 変化チップ安全装置
8. 安心して終了できる確認パネル
```

## 1. 行動理由ログ

AIキャラクターが行動した理由を短い文章で残す。

例。

```text
ジッピーは水分が低かったので飲み物を探した。
ロビンは体力が低かったので休憩した。
アルバートは社交的な特性でグリムに話しかけた。
マキナの保護イベントで病院送り前に警告が出た。
```

目的。

- AIキャラクターが何を見て行動したか分かるようにする。
- 観測HUD、生活ログ、放置復帰レポートへつなげる。
- 行動パッケージやOffMapActionの結果を説明しやすくする。

---

## 2. AIキャラクターの習慣

AIキャラクターが繰り返した行動を、少しずつ習慣として記録する。

例。

```text
よく朝にミルクを飲む → 朝ミルク習慣
よく同じ席に座る → お気に入り席
よく同じ相手と話す → 話し相手候補
よく同じ商品を買う → 常連購入品
```

目的。

- AIキャラクターの生活が積み上がっていく感覚を作る。
- 好き嫌い、特性、DailyWish、CustomerVisitReasonと連動する。
- プレイヤーが店づくりや家具配置を調整する理由になる。

---

## 3. 部屋・施設の雰囲気タグ

マップ、部屋、家具、施設に雰囲気タグを持たせる。

例。

```text
静か
にぎやか
清潔
退廃的
高級
安心
危険
香ばしい
機械的
神聖
```

使い道。

- AIキャラクターの居心地判定。
- 好き嫌いや特性による場所選び。
- お客としての来店理由。
- LifeEventDirectorのイベント補正。
- 店舗評価。

---

## 4. 放置復帰レポートの要約ランク

放置後のログをそのまま大量に出さず、重要度でまとめる。

候補ランク。

```text
重要
生活
店
関係
発見
注意
```

例。

```text
重要: ジッピーが体調注意
生活: ロビンが休憩して体力回復
店: 飲料がよく売れた
関係: アルバートとグリムの関係値が少し上昇
発見: 新しい好みが判明
```

---

## 5. 常連システム

AIキャラクターを単なる客ではなく、常連として育てられるようにする。

段階候補。

```text
初来店
気になる客
常連
お気に入り席あり
お気に入り商品あり
店員と仲がいい
特別注文をする
```

使い道。

- 来店理由の強化。
- 購入傾向の固定化。
- 会話イベントの追加。
- 店舗経営とAI生活の接続。

---

## 6. 小さな流行

街全体または施設ごとに、短期間の流行を持たせる。

例。

```text
今日は甘い飲み物が流行。
今週は清潔系雑貨が売れやすい。
雨の日は温かい料理が人気。
マキナの日は白い装飾品が少し人気。
ジッピーの影響で肉料理が話題。
```

使い道。

- 店舗経営の変化。
- 商品の需要補正。
- AI客の来店理由。
- LifeEventDirectorのイベント。

## 7. 変化チップ安全装置

これは必要仕様として採用する。

変化チップは、AIキャラクターの好き嫌い、特性、会話反応、購入傾向などに強いゲーム内補正を与える装備アイテムである。

そのため、プレイヤーが安心して扱える安全装置が必要。

必須UI。

```text
装備中チップ一覧
元設定との差分表示
取り外すと元に戻ることの表示
```

必須ルール。

```text
装備中の効果が見える。
元設定との差分が見える。
取り外せば元設定ベースに戻る。
相性が悪い場合は警告する。
強すぎる効果はマキナが安定性チェックする。
```

差分表示の例。

```text
元設定: ピザが苦手
装備中チップ: ピザ好み補正
現在の判定: ピザを好む
```

```text
元設定: 内向的
装備中チップ: 社交補正を追加
現在の判定: 会話行動を選びやすい
```

設計方針。

- 変化チップはキャラクター固有設定より優先度が高い。
- ただし、元設定を破壊しない。
- 元設定、チップ効果、最終判定を分けて扱う。
- UIでは「元設定」と「現在の補正後の判定」を見比べられるようにする。

将来の候補UI。

```text
AICharacterChangeChipPanel
AICharacterPersonalityDiffPanel
```

## 8. 安心して終了できる確認パネル

これは必要仕様として採用する。

放置、インクリメンタル、チル寄りのゲームでは、プレイヤーが安心してゲームを閉じられることも重要。

目的。

- 今閉じても大丈夫か分かる。
- 危険なAIキャラクターがいるか分かる。
- 店や在庫がどの程度もつか分かる。
- 次の大きな予定が分かる。
- 病院中AIの状態が安定しているか分かる。

表示例。

```text
今閉じても大丈夫。
次の大きな予定は3時間後。
危険なAIはいません。
在庫はあと半日もちます。
病院中のAIは安定しています。
```

注意がある場合。

```text
ジッピーの水分が低下しています。
飲料在庫が少なくなっています。
明日の朝に来店イベントがあります。
病院中のAIが退院条件に近づいています。
```

UI候補。

```text
SafeExitCheckPanel
```

Directorとの関係。

Deus Ex Machina Modeでは、マキナがやさしく状態を要約する。

例。

```text
マキナ: 今は安定しています。安心して休んでください。
マキナ: 水分補給の備えだけ、少し気にしておくとよさそうです。
```

## 実装優先度

コード作業を再開する場合の優先度。

```text
1. ActionRunner / 行動パッケージの続き
2. AI観測HUDの改善
3. 行動理由ログ
4. LifeLog
5. ReportUI
6. SafeExitCheckPanel
7. 変化チップ安全装置
8. 習慣、雰囲気タグ、常連、小さな流行
```

変化チップ安全装置は、変化チップ本体を作る前に必ず設計する。

SafeExitCheckPanelは、放置・チル方針の核として早めに検討する。
