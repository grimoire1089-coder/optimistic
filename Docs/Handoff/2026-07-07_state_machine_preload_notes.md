## 動画内容メモ: ステートマシンの活用場面

動画の制作者は、Godot を使った農業シミュレーションゲームの開発において、「ステートマシン（現在の状態を管理し、条件に合わせて切り替える仕組み）」を大きく 3 つの場面で活用している。

### 1. ゲーム全体の画面遷移や進行管理

- タイムスタンプ: `[00:34]`
- スクリプトベースのステートマシンを使用して、ゲーム全体の進行状態を管理している。
- ステートマシンはキャラクター制御に使われることが多いが、ゲーム全体のループ管理にも役立つ。

具体例:

```text
スプラッシュスクリーン
↓
メインメニュー
↓
ゲームプレイ
```

新規ゲーム開始時の具体例:

```text
メインメニュー
↓
イントロシーン（背景説明のテキストスクロール）
↓
ゲームプレイ（ワールドの読み込み）
```

### 2. 農場の動物の行動ルーチン

- タイムスタンプ: `[06:01]`
- ノードベースのステートマシンを使用して、動物たちの毎日の行動スケジュールを制御している。

具体例:

```text
朝
  ニワトリが小屋を出て歩き回る状態へ切り替わる。

夜
  ニワトリが自動的に小屋へ帰る状態へ切り替わる。
  経路探索して帰る。
```

### 3. 敵キャラクターのAI

- タイムスタンプ: `[07:08]`
- ニワトリ用に構築したステートマシンのロジックを応用・調整して、敵キャラクター（スライム）のAIを作成している。

具体例:

```text
Wandering / 徘徊する
Chasing / プレイヤーを追いかける
Fleeing / 逃げる
```

状況に応じて、スライムが異なる状態を動的に切り替えながら行動する。

## 動画内容メモ: カレンダーと天気の事前生成システム

### 1年分の予定を先回り作成

- ゲームがロードされた瞬間に、カレンダーイベントマネージャーが 1年分のフェスティバルや季節ごとの天気をあらかじめ生成する。
- この考え方はプリロール、事前生成、事前計算として使える。

### シグナルでの連携

- 「時間が進んだ」というシグナルを合図に、雰囲気マネージャーがカレンダーの予定をチェックする。
- 夕方になったら画面を暗くする。
- 予定通りの雨を降らせる。

## 動画内容メモ: オートロードの最小化

### 必要なときだけ読み込む

- どこからでもアクセスできるオートロード、常駐スクリプトは最小限にする。
- 天候管理を常駐にした結果、メインメニューに雨が降る大失敗があったとのこと。

### ワールドシーンの子ノードにする

- ほとんどの管理システムは、ゲームプレイが始まった瞬間に読み込まれるメインワールドの子ノードとして配置するように修正した。
- 常に存在させる必要がないものは、Autoload ではなくワールド配下に置く。

## 動画内容メモ: JSONデータとカスタムリソースの合体

### データ管理の効率化

- Godot にはカスタムリソースがある。
- ただし、アイテムなどを何百個もインスペクターで手作業作成するのは面倒。

### 自動変換ツールを自作

- 1つの JSON ファイルからデータを読み込む。
- 実行時に自動で Godot のカスタムリソースへ変換する。
- 鉱石やアイテムを増やすのが一瞬でできるようになる。

## 動画内容メモ: 1日の終わりの裏方処理

### 作物の成長

- プレイヤーがベッドで寝ると、裏でワールドマネージャーが動く。
- マップ上の耕されたタイルをすべて見回る。
- 水が撒かれていたら作物の成長段階を1歩進める。

### ゴミや障害物の自動拡散

- 手入れされていないエリアに、石、丸太、雑草などの瓦礫がランダムで確率発生して広がる計算をする。

## 動画内容メモ: オーディオバスによる臨場感

### 洞窟のリバーブ

- 炭鉱、マインに入ったときの音の響きをリアルにする。
- Godot のオーディオバス効果を使う。
- 広い空間特有のエコー、残響をかける。

## 動画内容メモ: NPCのスケジュール管理

### テキストから行動を読む

- 町の住民、NPC の1日の行動ルーチンも JSON ファイルに書かれたスケジュールデータを読み込ませて制御している。

## optimistic に持ち込むときの注意

このメモは、すぐ修正するための実装指示ではなく、今後の設計相談用の記録。

現時点では以下を優先して考える。

1. 図鑑メッセージ付近の FPS 低下を調べる。
2. ステートマシンは大規模導入ではなく、必要な箇所から小さく導入する。
3. 事前ロードは、常に必要な軽量リソースに限定する。
4. Autoload は増やしすぎず、ワールドやUIシーンの子ノードとして置ける管理システムはそちらを優先する。
5. JSON とカスタムリソースの合体は、アイテム、レシピ、NPCスケジュール、カレンダー、天気などに応用できる可能性がある。
6. 就寝時、日付変更時、画面遷移中など、プレイヤーが納得しやすいタイミングに重い処理を寄せる。

## 今回のFPS低下と関係しそうな観点

図鑑メッセージ付近で FPS が下がるため、次に調べる候補は以下。

- 図鑑解放通知のメッセージ生成。
- メッセージログカードの生成。
- StyleBoxFlat や Label など UI ノードの毎回生成。
- 通知SFXの初回ロード。
- 後追いで図鑑メッセージを探して装飾する処理。
- 同じフレームに複数の UI 更新、ログ追加、音再生、スクロール処理が集中していないか。

## 方針メモ

- 修正はまだ行わない。
- まずステートマシン、事前ロード、常駐システムの責任範囲を話し合う。
- 次に直す場合も、関係ないコードは触らず、図鑑通知やメッセージログなど原因箇所に絞る。
